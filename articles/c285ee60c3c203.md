### fierce

fierceは、ドメインからIPアドレスの範囲を見つけ出し、その周辺にもターゲットが隠しているサーバーが無いかを探します。

ワードリストを用いてサブドメイン名を片っ端から試した後、ヒットしたIPアドレスがあった場合、その同じネットワーク内に存在する「名前が登録されていない（あるいは別の名前の）サーバー」を、逆引きを使って炙り出す。

これにより、メインのWebサイトとは全く別の場所に置かれている「忘れ去られた開発サーバー」などを見つけ出す。

::: message
例えば、辞書攻撃によって dev.sample.com が 1.2.3.100 だと判明した場合、fierceは追加の命令がなくても、1.2.3.1 から 1.2.3.254 までのすべてのIPアドレスに対して「名前は何ですか？（逆引き）」と聞きに回ります。
:::

基本的な使い方は以下
```
fierce --domain sample.com
```
よく使うオプション

- --domain: 対象のドメイン（必須）
- --wide: 見つかったサブドメインとどういうクラスのネットワーク全体をスキャン（重要）
- --subdomains-file: 使用する辞書ファイル


サイバーキルチェーンの「偵察」において使用されます。

| 偵察 | 武器化 | 配送 | 攻撃 | インストール | 遠隔操作 | 目的達成 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 〇 |  |  |  |  |  |  |

::: message
サイバーキルチェーンとは、攻撃者がサイバー攻撃を成功させるまでの一連の流れを体系化したモデルです
:::

### 実行結果の見方

#### DNSサーバー

以下のDNSサーバーが見つかったことがわかります
```
[+] DNS Servers for sample.com:
    ns1.sample.com (1.2.3.10)
    ns2.sample.com (1.2.3.11)
```
---

#### ゾーン転送

ゾーン転送は失敗したことがわかります
```
[+] Trying zone transfer on ns1.sample.com...
    [!] Exactic zone transfer failed. (Refused)
```

#### ブルートフォース

ブルートフォース攻撃によって以下のサブドメインが見つかりました
```

    1.2.3.100    admin.sample.com
    1.2.3.101    dev.sample.com
    1.2.3.105    mail.sample.com
    13.230.1.5   www.sample.com

```

#### 周辺のIPアドレス探索

見つかったIPアドレスの周辺を探索して得られたIPアドレスです。
fierceの特徴的な挙動です。
周辺を検索した結果全く関係ないドメインが出た場合は、クラウドなどの共用サーバーなどの可能性が考えられます。

```
[+] Searching for nearby IPs...
    1.2.3.98     (Found via reverse DNS) -> internal-wiki.sample.com
    1.2.3.99     (Found via reverse DNS) -> staging.sample.com
    1.2.3.102    (Found via reverse DNS) -> printer-off-tokyo.sample.com
```
---

#### ドメイン名のないもの

###　結果の活用

メールサーバの種類やバージョンから、脆弱性が見つかることがあります。
また、オープンリレーの設定が許可されていると、攻撃者は迷惑メールの踏み台として利用できます。
他にも、DKIM/SPF/DMARCが実装されていない場合、なりすましによるフィッシングメール攻撃の成功率が上がります。
取得したIPアドレスでターゲットに対してポートスキャンをすることで、公開されているサービスやバージョンを特定します。古いバージョンであれば脆弱性が見つかることがあります。
また、IPアドレスからドメインを逆引きし、一つのIPアドレスに複数のドメインが紐づいていれば、新たな攻撃対象になります。
サブドメインを列挙することで、古いアプリケーションやテスト環境、放置された脆弱なサーバが見つかることがあります。

## まとめ

DNS Analysis（DNS分析）に関しては、詳細な分析をする場合は設定ミスの検出や、様々の出力形式に対応しているdnsreconを使用し、周辺のIPまで含め徹底的に攻撃対象を洗い出したい場合はfierceを使うとよさそうです。
