## ライフサイクル

データのライフサイクルとは、データが生成または収集されてから、最終的に破棄されるまでの一連の流れのことを表す。

| ステップ | 概要 |
| ---- | ---- |
| 生成・収集 | ログ、ユーザーの入力、外部APIなどからデータが作成または取得される |
| 格納・保管 | 収集したデータをデータベース、クラウドなどに保存 |
| 利用 | アプリケーションでの処理や、分析のためにデータが読み込まれ加工される |
| 保持・共有 | 業務利用から外れた場合でも、将来的な監査などのために長期保持する |
| 破棄 | 保持期間が過ぎた場合や、不要になったデータを復元不可能な状態にする |

## 役割

### データオーナー

データオーナーは、そのデータに対して「最終的な法的責任」を持つ個人である。
情報セキュリティのCIA（機密性、完全性、可用性）を確保できるような対策を実施する。
通常は社長、経営幹部、部門長等のマネジメント層がその役割にあたる。

:::message
ライフサイクルに従って正しくデータを扱えていない場合、どのデータが最新で正しく、どのデータが古く不要なのか判断ができなくなる。そうなってしまえば、正しいデータが含まれていたとしても使えないデータとなってしまう。
:::
### データ管理者

データ管理者は、データオーナーが決定してポリシーに従い、データを保護するために日々、維持・管理を行う実務者のことである。
悪性制御の実装、パッチ、システムのアップデート、バックアップやリストアなどを行う。

### 比較まとめ
| 項目 | データオーナー | データ管理者 |
| ---- | ---- | ---- |
| 主な役割 | ポリシーの決定・責任 | 技術的な実装・運用 |
| 担当者 | CEO、部門長等 | システム管理者、ITスタッフ等 |

:::message
データオーナーはあくまでそのデータの保護責任者であり、データの持ち主はユーザーである。
:::

### GDPRにおける役割
GDPR（欧州一般データ保護規則）においては、さらに厳密な役割が定義されており「データコントローラー」と「データプロセッサー」が存在する。

:::message
組織がデータをクラウド上で管理する場合、データを扱う組織側を「データコントローラー」、データを管理・処理しているクラウド側を「データプロセッサー」とよぶ。
:::

## データの収集

データの収集には、市場調査やアンケートなどの用途が明確になった上で収集するデータもあれば、工場などのセンサーから取得されたログなど、用途は明確でないが、将来的に役立つかもしれないということで収集されているデータが存在する。

:::message
一見不要だと思われるデータも、将来的に重要なデータとなりうることもあるため、簡単には破棄せずに長期間保有することもある。
:::

### データの収集方法

データ収集の例として以下のようなものがある

| 項目 | 概要 |
| ---- | ---- |
| ユーザー入力 | Webサイトの問い合わせフォームや、アカウント登録、アンケートなどから収集 | 
| 自動ログ収集 | Webサーバのアクセスログや認証ログなどから取得 |
| センサー・IoTデバイス | 工場のセンサーや防犯カメラなどから取得 |
| Webスクレイピング | 公開されているWebサイトから自動情報抽出（スクレイピング）を使用して取得 |
| API | 公開されているWebサイトが提供するAPIを使用して取得 |

:::message
ユーザー入力によるデータ収集では、SQLインジェクション、XSSなどの攻撃コードを仕込まれるリスクがあるため対策が必要
:::

:::message
スクレイピングが禁止されているWebサイトもあるため注意が必要
:::

![](https://static.zenn.studio/user-upload/8a3c80032896-20260706.png)

### データの管理場所

データは、オンプレミスサーバーや、クラウドなど様々な場所に保管される。
データマッピングによって、どこに何のデータがあるのかを明確にし、安全に管理する必要がある。

:::message
近年の個人情報保護法（改正APPI）やGDPRなどの法規制では、個人データが「どこの国のサーバーに保管されているか」「委託先（サブプロセッサー）がどの地域で処理しているか」の明示を求めている。
データマッピングは、技術的な安全対策（セキュリティ）のためだけでなく、法的な義務（コンプライアンス）を果たすための必須要件となっている。
:::

## まとめ

- データオーナーはデータの責任者であり、データ管理者は実務者である
- ユーザー入力、API環境庁の資料など様々なデータ収集方法がある
- データの所在管理は、安全性だけでなくコンプライアンスを満たすことにもつながる
