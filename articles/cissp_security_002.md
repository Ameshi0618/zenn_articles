## セキュリティとは

守るべき資産を、あらゆるリスクから守り続けることである。
リスクから資産を守るために実行する防御策のことをセーフガードという。
このセーフガードを考え、実行していくことこそがCISSPの仕事である。

![](https://storage.googleapis.com/zenn-user-upload/54b1c297f3cf-20260117.png)



## CIAトライアド
セキュリティを確保するのに必要な3つの要素である。

| 要素 | 説明 | 技術例 | 侵害例 |
| ---- | ---- | ---- | ---- |
| 機密性（Confidential） | 許可されたものだけが情報にアクセスできること | 暗号化、アクセス制御、多要素認証 | 情報漏洩、ソーシャルエンジニアリング |
| 完全性（Integrity） | 情報が正確で、改ざんされていないこと | デジタル署名、バージョン管理 | Webサイトの改ざん、データの書き換え |
| 可用性（Availability） | 許可された者が、いつでも情報を利用できること | 冗長化、バックアップ | システムダウン、ランサムウェア、DDos攻撃 |

:::message
この3要素のうちどれが一番優先すべきかという疑問が生じる。一般的なビジネスにおいては機密性が最優先となる。しかし、OT（産業）業界や医療業界では可用性が最優先となる。機密性を守るために認証を強化した結果、緊急時にシステムが停止し人の命が亡くなってしまうからだ。
:::

3要素のバランスを取りながら、すべての要素を強固にしていくことが重要である。
例えば、スマホの顔認証は、開くたびに認証する必要があるが、パスワード入力の手間が省けるため、3要素全てがバランスよく強固になったと言えるだろう。

:::message
逆にバランスを取らず、機密性と完全性だけ強固にするのであれば、毎秒多要素認証を強要すればよい。しかし、可用性が低すぎてサービスとして使い物にならない。そうなると、ユーザーが可用性が上げるためにパスワードを簡単にするなどをしてしまい、最終的に機密性と完全性の低下に繋がってしまう。
可用性は一見セキュリティに関係ないが、セキュリティ対策ができている使いやすいサービスを考える上で必要な観点である。
:::

### DAD
セキュリティを確保するのに、**やってはいけない**3つの要素である。

| 要素 | 説明 | 対応するCIAの要素 |
| ---- | ---- | ---- |
| 漏洩（Disclosure） | 本来見せてはいけない情報が漏れること | 機密性 |
| 改ざん（Alteration） | データや設定が不正に書き換えられること | 完全性 |
| 破壊（Denial） | 情報やシステムが使えなくなること | 可用性 |

DADはCIAを破壊する要素と言い換えられる。

## 多層防御

セキュリティを考える上でセーフガードは多層であるべきという考えである。

例えばファイアウォールを設置することで不正な通信を可能な限り監視することは可能だが、実際に侵入された場合の対策としてEDRなどを導入することが重要である。

:::message
自宅のセキュリティが玄関の鍵1つではなく、オートロックや監視カメラを設置することも多層防御と言えるだろう。
:::

## 情報保護の種類

曖昧さによる保護は、偶然見つかってしまったりすることも考えられるため理想的ではない。

| 種類 | 説明 | 例 | 評価 |
| ---- | ---- | ---- | ---- |
| 曖昧さ(obscurity) | 情報が存在していることがわからないようにすること | 機密情報にはだれでもアクセスできるがその場所は誰にも教えていないから大丈夫 | Bad |
| 隠蔽(data hiding)） | 情報が存在していることはわかっても、詳細な情報を取得することが困難なこと | パスワードがあればアクセスできることが分かったが、パスワードがわからない | Good |

## まとめ

- CIAはバランスよく強固にしていくためのセーフガードを考えることが重要である
- セーフガードは単一ではなく、複数の対策を重ねる多層防御が重要である
- セーフガードは仕組みがわかっても突破できないのが理想
