### lbd

対象のドメインがロードバランサーを使用しているかどうか確認します

基本的な使い方は以下
```
lbd sample.com
```

サイバーキルチェーンの偵察において使用されます。

| 偵察 | 武器化 | 配送 | 攻撃 | インストール | 遠隔操作 | 目的達成 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 〇 |  |  |  |  |  |  |

::: message
サイバーキルチェーンとは、攻撃者がサイバー攻撃を成功させるまでの一連の流れを体系化したモデルです
:::

#### DNS問い合わせ

DNSクエリを複数回投げることで変えってくるIPアドレスが同じかどうかを確認します。
同じであればロードバランサーを使用していないことわかります。

2つのIPアドレスが返ってきたのでロードバランがーが使用されていることがわかります
```
Checking for DNS-Loadbalancing: FOUND
93.184.216.34
93.184.216.35
```

lbdによる確認はDNSラウンドロビンのロジックを利用したものです。DNSラウンドロビンの場合は、問い合わせの旅に異なるIPアドレスが返ってきます。

::: message
単純に複数のIPアドレスを順番に割り振るので、どこか1つのサーバーに負荷がかかっていても順番に割り振るので負荷分散できない。
:::

Anycast IPを使用している場合は、外部から見たら1つのIPアドレスに見えます。

::: message
Anycastはある１つのIPアドレス宛に送ると、その先では複数のIPアドレスに紐づいていて、そのうち1番近いところに転送してくれるIPアドレスです。負荷分散するという意味では優れています。
:::

---

#### Serverフィールド

HTTPレスポンスヘッダーのServerフィールドを確認します

複数のサーバーが使用されていることがわかります
```
Checking for HTTP-Loadbalancing [Server]: 
 nginx/1.18.0
 Apache/2.4.41
 FOUND
 ```

---

#### Dateヘッダー

HTTPレスポンスのDateヘッダーを見て、時刻が同期されているか確認します

時刻が同期されていないので複数のサーバーが使用されています
```
Checking for HTTP-Loadbalancing [Date]: 14:10:05, 14:10:07, 14:10:05, 14:10:07, FOUND
```

---

#### レスポンスのコンテンツ

HTTPレスポンスの中身を見て、内容に差分が無い確認します

差分が無いため、HTTPレスポンスの中身では判断できなかったようです
```
Checking for HTTP-Loadbalancing [Diff]: NOT FOUND
```

::: message
サーバーによっては毎回ランダムな値が生成されている可能性があり、それを見て中身が異なるからロードバランサーがあると誤検知する可能性はあります。中身のどこに違いがあったのかは注意したほうがよさそうです。
:::

---

#### 結果

DNS HTTP[Server] HTTP[Date]を確認した結果、ロードバランサーを使用している可能性が高い、ということがわかりました。
ヘルプの説明文に誤検知の可能性があるという記載もある通り、実行結果は参考にとどめておくのがよさそうです。

```

```
