### arping

対象ホストにARP要求およびICMP要求を送信し応答を確認します。
同一ネットワーク内でのみ使用できます。

::: message
pingだとファイアウォールでブロックされる可能性があっても、ARP要求はデータリンク層の通信のためブロックされず、確実な生存確認ができます
:::

基本的な使い方は以下
```
sudo arping -i eth0 192.168.X.X
```

よく使われるオプション

- -i: 使用するネットワークインターフェースを指定。指定が無い場合は自動で決まります。
- -c: 自動的に停止させたい場合は送信回数を指定
  

サイバーキルチェーンの「偵察」において使用されます。

| 偵察 | 武器化 | 配送 | 攻撃 | インストール | 遠隔操作 | 目的達成 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 〇 |  |  |  |  |  |  |

::: message
サイバーキルチェーンとは、攻撃者がサイバー攻撃を成功させるまでの一連の流れを体系化したモデルです
:::

::: details ヘルプ
```
$ arping --help
ARPing 2.25, 作者: Thomas Habets <thomas@habets.se>

使用法:
  arping [ -0aAbdDeFpPqrRuUvzZ ] [ -w <秒> ] [ -W <秒> ] [ -S <ホスト/IP> ]
         [ -T <ホスト/IP> ] [ -s <MAC> ] [ -t <MAC> ] [ -c <回数> ]
         [ -C <回数> ] [ -i <インターフェース> ] [ -m <タイプ> ]
         [ -g <グループ> ] [ -V <VLAN> ] [ -Q <優先度> ]
         <ホスト/IP/MAC | -B>

オプション:

  -0     送信元 IP アドレス 0.0.0.0 で ARP ping を行う。
         インターフェースに IP を設定していない場合に使用する。
         この方法では MAC-ping が応答されないことがある。
         -S 0.0.0.0 の別名。

  -a     応答があったときに音を鳴らす。

  -A     要求したアドレスに一致する応答のみをカウントする。
         （これは多くの動作を壊す *可能性がある*）
         複数ホストを同時に arping する場合にのみ有用。
         例は arping-scan-net.sh を参照。

  -b     -0 と同様だが、送信元アドレスにブロードキャスト
         (255.255.255.255) を使用する。
         通常の挙動ではないため、応答されないことがある。

  -B     ホスト指定の代わりに使用し、255.255.255.255 を対象とする。

  -c 回数
         指定した回数だけ ARP リクエストを送信する。

  -C 回数
         -c や -w に関係なく、指定した数の応答を受け取ったら終了。

  -d     重複応答を検出する。
         異なる 2 つの MAC アドレスから応答があった場合、終了コード 1。

  -D     応答があれば「!」、応答がなければ「.」を表示。

  -e     -a と同様だが、応答がない場合に音を鳴らす。

  -F     インターフェース名を自動判別しない。
         （-i を指定した場合は常にこちらが優先される）

  -g グループ
         nobody グループの代わりに、このグループへ setgid() する。

  -h     ヘルプメッセージを表示して終了。

  -i インターフェース
         使用するネットワークインターフェースを指定。

  -m タイプ
         受信パケットに使用するタイムスタンプの種類。
         -vv を付けると利用可能なタイプを表示。

  -q     エラーメッセージ以外を表示しない。

  -Q 優先度
         802.1p の優先度を設定する。
         802.1Q (-V) と併用すること。
         デフォルトは 0。

  -r     生出力モード。
         各応答について MAC/IP アドレスのみを表示。

  -R     生出力モード。
         -r と似ているが「もう一方」を表示。
         -r と併用可能。

  -s MAC
         送信元 MAC アドレスを指定。
         この場合、-p が必要になることがある。

  -S IP
         -b や -0 と同様だが、送信元 IP を指定する。
         対象ホストがその IP への経路を持たない場合、
         応答されないことがある。
         自分が所有していない IP を使う場合、
         -p でプロミスキャスモードを有効にする必要がある。
         このオプションを使うことで、自分が IP を使用せずに
         相手ホストの IP アドレスを調べることができる。

  -t MAC
         IP アドレスに対して ping を行う際の対象 MAC アドレスを指定。

  -T IP
         ブロードキャスト ARP に応答しないが、
         指向性ブロードキャストには応答する可能性がある
         MAC に対して使用するターゲット IP。

         例:
         MAC-A のアドレスを確認するために、
         MAC-B と IP-B が分かっている場合:
         $ arping -S <IP-B> -s <MAC-B> -p <MAC-A>

  -p     インターフェースをプロミスキャスモードにする。
         自分が「所有していない」MAC アドレスを使う場合に必要。

  -P     ARP リクエストではなく ARP リプライを送信。
         -U と組み合わせて使用すると有用。

  -u     MAC を ping する際、
         index=受信/送信 の形式で表示する。

  -U     Unsolicited ARP（要求されていない ARP）を送信。

  -v     詳細表示。
         2 回指定するとさらに詳細を表示。

  -V 数値
         802.1Q VLAN タグを追加する。
         デフォルトは VLAN タグなし。

  -w 秒
         送受信回数に関係なく、指定時間後に終了。

  -W 秒
         ping の送信間隔。

  -z     seccomp を有効化。

  -Z     seccomp を無効化（デフォルト）。

バグ報告先: thomas@habets.se
Arping ホームページ: http://www.habets.pp.se/synscan/
開発リポジトリ: http://github.com/ThomasHabets/arping

```
:::

### 実行結果の見方

#### 対象のMACアドレス

MACアドレスが00:0c:29:43:a1:feであることがわかりました
```
42 bytes from 00:0c:29:43:a1:fe (192.168.X.X): index=0 time=1.234 msec
```

---

#### タイムアウト

指定したIPアドレスは存在しないため応答が得られませんでした
```
Timeout
```

### 結果の活用

ネットワークに侵入した後、どのホストが生存しているか確認するのに使用できます。

::: message
-0 や -T オプションを使用することで、送信元のIPアドレスを指定することができます。これにより、自分のIPアドレスを偽装して、対象の生存確認ができます。
:::
