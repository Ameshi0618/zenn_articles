## 所有者と管理者

情報や資産には、所有権を割り当てる必要がある。
所有権を割り当てることで責任の所在を明確にすることが重要である。

:::message
責任を明確にすることで、インシデント発生時に迅速な意思決定が可能になります。
:::

![](https://static.zenn.studio/user-upload/50b77d57ed96-20260704.png)

| 分類 | 説明 |
| ---- | ---- |
| データ所有者 | データのビジネス上の価値を理解し、データの分類や保護要件を決定する |
| データ管理者 | データ所有者が決定したポリシーや分類に従い、アクセス権の設定などを行う。 |

## 資産インベントリ

資産インベントリとは、組織が持つすべての電子的及び物理的な情報や資産をリストにして管理する仕組みのことである。
物理的なパソコンや、サーバーだけでなく、電子的な資産であるファイルやライセンスなども含まれる。

:::message
「見えないものは守れない」ため、資産インベントリはセキュリティの基礎とも言える
:::

資産の価値、状態、管理場所、バージョン、所有権の所在等の情報を管理することで、セキュリティリスクを低減し、効率的な運用が可能になる。

全ての管理を手動で行うことは難しいため、資産管理のソフトウェアや、物理的な資産にRFIDなどを使用することも重要である。

:::message
使用されていないパソコン、サーバーの処分や、使用されていないライセンスの停止をすることでコストカットに繋げることが可能である
:::

### 電子的資産と物理的資産の紐づけ

電子的資産と物理的資産は独立しているとは限らない。
使い古したサーバーの物理的な価値は数万円程度しかないかもしれないが、そのサーバーの中に「機密情報」や「顧客情報」などが含まれているのであれば、このサーバーには数十億円の価値がある。

そのため、どの物理的資産の中にどんな電子的資産があるのかの紐づけが重要なのである。

### 防ぐことができるリスク

資産インベントリを作成することで以下のようなリスクを防ぐことが可能

| リスク | 説明 |
| ---- | ---- |
| シャドーIT | 組織が把握できていないサーバーやデバイスがあると、そこがセキュリティホールとなる |
| 脆弱性の管理不足 | 新たな脆弱性が発見された際に、自社に影響があるかどうかの判断が遅れる |
| 不適切なリソース配分 | 重要ではない資産にコストをかけすぎてしまったり、重要な資産が放置されてしまう |

## まとめ

- 所有者を明確にすることで、責任の所在を明確にする
- 資産インベントリを作成することで、セキュリティリスクを減らし、効率よく資産管理、活用が可能になる
