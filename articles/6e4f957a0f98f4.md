### dnsrecon

dnsreconは、dnsenumやdnsmapよりも後に開発された多機能なDNS情報収集ツールです。より現代的なプロトコルや高度な調査手法に対応しています。

::: message
単なるサブドメインの探索だけでなく、DNSの構成ミスやセキュリティ不備（DNSSECやレコードの不整合）を診断する機能を持ちます。
:::

基本的な使い方は以下
```
dnsmap -d sample.com
```
よく使うオプション

- -d: 対象のドメイン（必須）
- -D: 使用する辞書ファイル
- -t: スキャンタイプ

サイバーキルチェーンの「偵察」において使用されます。

| 偵察 | 武器化 | 配送 | 攻撃 | インストール | 遠隔操作 | 目的達成 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 〇 |  |  |  |  |  |  |

::: message
サイバーキルチェーンとは、攻撃者がサイバー攻撃を成功させるまでの一連の流れを体系化したモデルです
:::

::: details ヘルプ
```
$ dnsrecon -h     
使用法: dnsrecon [-h] [-d ドメイン] [-n ネームサーバ] [-r 範囲] [-D 辞書ファイル] [-f] [-a] [-s] [-b] [-y] [-k] [-w] [-z] [--threads スレッド数] [--lifetime 応答待機時間] [--tcp] [--db ファイル名] [-x ファイル名] [-c ファイル名] [-j ファイル名] [--iw]
                [--disable_check_recursion] [--disable_check_bindversion] [-V] [-v] [-t タイプ]

オプション:
  -h, --help            ヘルプを表示して終了。
  -d, --domain DOMAIN   ターゲットとなるドメイン。
  -n, --name_server NS_SERVER
                        使用するネームサーバーを指定。未指定の場合はターゲットのSOAレコードを使用。カンマ区切りで複数指定可能。
  -r, --range RANGE     逆引きルックアップをブルートフォースで行うための IP アドレス範囲。
                      指定形式は (開始IP-終了IP) または (範囲/ビットマスク)。
  -D, --dictionary DICTIONARY
                        総当たりに使用するサブドメイン／ホスト名の辞書ファイル。
  -f                    保存時、ワイルドカード設定されたIPに解決されるレコードをフィルタリング（除外）する。
  -a                    標準的な列挙に加え、AXFR（ゾーン転送）を試行する。
  -s                    標準的な列挙に加え、SPFレコード内のIPv4範囲を逆引きする。
  -b                    標準的な列挙に加え、それぞれ Bing を使用して列挙。
  -y                    標準的な列挙に加え、それぞれ Yandex を使用して列挙。
  -k                    標準的な列挙に加え、それぞれ crt.sh を使用して列挙。
  -w                    標準的な列挙に加え、Whoisレコードの深い分析と発見されたIP範囲の逆引きを実行。
  -z                    標準的な列挙に加え、DNSSECのゾーンウォークを実行。
  --threads THREADS     並列処理（スレッド）数。逆引き、正引き、総当たり等に使用。
  --lifetime LIFETIME   応答待機時間（デフォルトは3.0秒）。
  --tcp                 クエリにTCPプロトコルを使用する。
  --db DB               結果を SQLite 3 ファイルに保存。
  -x, --xml XML         結果を XML ファイルに保存。
  -c, --csv CSV         結果を CSV ファイルに保存。
  -j, --json JSON       結果を JSON ファイルに保存。
  --iw                  ワイルドカードレコードが発見されても総当たりを続行する。
  --disable_check_recursion
                        ネームサーバに対する再帰可否のチェックを無効にする
  --disable_check_bindversion
                        ネームサーバに対する BINDのバージョンチェックを無効にする
  -V, --version         dnsreconのバージョンを表示する。
  -v, --verbose         詳細表示を有効にする
  -t, --type TYPE       実行する列挙（調査）のタイプを指定
                        利用可能なタイプ:
                            std:      SOA, NS, A, AAAA, MX, SRV レコード
                            rvl:      指定した CIDR または IP 範囲に対する逆引きルックアップ
                            brt:      指定した辞書を用いてドメイン名・ホスト名をブルートフォース
                            srv:      SRV レコードの列挙
                            axfr:     すべての NS サーバに対してゾーン転送（AXFR）をテスト
                            bing:     Bing 検索を利用してサブドメインやホストを列挙
                            yand:     Yandex 検索を利用してサブドメインやホストを列挙
                            crt:      crt.sh を利用してサブドメインやホストを列挙
                            snoop:    指定したドメインに対して、すべての NS サーバへ
                                      DNS キャッシュスヌーピングを実施する。
                                      -D オプションで指定したファイル内のドメインを使用
                        
                            tld:      指定したドメインから TLD を除去し、
                                      IANA に登録されているすべての TLD と照合する
                            zonewalk: NSEC レコードを利用した DNSSEC ゾーンウォークを実行

```
:::

### 実行結果の見方

#### SOAレコード

指定したドメインを管理している権威DNSサーバが表示されます。
```
[*] 	 SOA ns1.example.com 93.184.216.34
```
---
#### NSレコード

指定したドメインを管理しているネームサーバーが表示されます。
```
[*] 	 NS ns1.example.com 93.184.216.34
[*] 	 NS ns2.example.com 93.184.216.35
```
---
#### MXレコード

指定したドメイン宛てのメールを管理しているメールサーバーが表示されます
```
[*] 	 MX mail.example.com 93.184.216.40 10
[*] 	 MX backup.example.com 93.184.216.41 20
```
---
#### Aレコード

指定したドメイン自体のIPアドレスが表示されます。
```
[*] 	 A example.com 93.184.216.34
```
---
#### SRVレコード

どのサーバで、何のサービスを使用しているかを表示します
```
[+] 	 SRV _sip._tcp.example.com sipserver.example.com 93.184.216.50 5060
```

::: message
取得できたサブドメインの利用方法については、dnsenumの記事をご覧ください
::: 

---
#### 設定不備の指摘

調査した結果SPFレコードの等の設定に不備があると以下のように教えてくれます。

```
[*] 	 TXT example.com v=spf1 ip4:1.2.3.4 include:_spf.google.com +all
[!] SPF record allows any host to send email for this domain: +all
```

### 結果の活用

メールサーバの種類やバージョンから、脆弱性が見つかることがあります。
また、オープンリレーの設定が許可されていると、攻撃者は迷惑メールの踏み台として利用できます。
他にも、DKIM/SPF/DMARCが実装されていない場合、なりすましによるフィッシングメール攻撃の成功率が上がります。
取得したIPアドレスでターゲットに対してポートスキャンをすることで、公開されているサービスやバージョンを特定します。古いバージョンであれば脆弱性が見つかることがあります。
また、IPアドレスからドメインを逆引きし、一つのIPアドレスに複数のドメインが紐づいていれば、新たな攻撃対象になります。
サブドメインを列挙することで、古いアプリケーションやテスト環境、放置された脆弱なサーバが見つかることがあります。

dnsenum、dnsmap、dnsreconの3つの内、**dnsrecon**を使用しておけばひとまず機能としては網羅できそう。
