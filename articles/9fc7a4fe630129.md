## ベンダー管理

システムを構築するにあたり、自社で1から100まですべて完結していることはほぼあり得ない。そこで、特定の業務などを外部に委託し、自社の期待どおり・安全に・継続して価値を提供できるように管理する。

![](https://storage.googleapis.com/zenn-user-upload/287332b9aaa0-20260205.png)

### ベンダー管理のライフサイクル

| フェーズ | 主な活動 |
| ---- | ---- |
| ➀選定 | 財務状況、実績、**セキュリティ体制の確認** |
| ➁契約 | 責任範囲の定義、監査権限の明文化 |
| ➂監視 | 定期的な監査、インシデント報告の確認 |
| ➃終了 | データの返却、権限の剥奪 |

:::message
委託先で情報漏洩などが発生した場合、委託元である自社のセキュリティ管理不足となってしまう。そのため、委託先のセキュリティ体制などは十分に確認する必要がある。
:::

### マルチベンダー戦略

マルチベンダー戦略とは、特定のベンダーに依存せず、複数のベンダーを併用することでコスト・リスク・供給停止の影響を分散する戦略である。
事業継続計画においても非常に重要な概念である。

- サプライチェーンの停止: ベンダーがサイバー攻撃を受けたり、倒産した場合に共倒れしてしまう。
- ベンダーロックイン: 他のベンダーへの乗り換えが困難になり、法外な値上げや、品質低下を受け入れざるを得なくなる。
- 地政学的リスク：特定の国のベンダーを使用している場合、法律の変化により規制などにより、突然使えなくなることがある。

## まとめ

- ベンダー選定において、セキュリティ体制などを確認することが重要である
- 1つのベンダーに依存することはリスクである
