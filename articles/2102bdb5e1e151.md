### wafw00f

HTTPリクエストをを送りその応答から、使用されているWAF（WEBアプリケーションファイアウォール）を識別します。
判別できない場合は、攻撃的なリクエストを送り、その挙動やエラーの特徴から製品を特定します。

::: message
システムやアプリケーションの固有の特徴から製品を特定するこを「フィンガープリント」（指紋採取）と言います。
:::

基本的な使い方は以下
```
wafw00f http://www.sample.com
```

よく使われるオプション

- -v, -vv: 詳細な表示
- -a: 複数の製品の使用を考慮して全て調査
- -l: 検知できる製品が知りたいとき
  
::: message
オプション-Hを使用して、User-Agentを偽装しブラウザのふりをすることで接続拒否を回避します
:::

サイバーキルチェーンの偵察において使用されます。

| 偵察 | 武器化 | 配送 | 攻撃 | インストール | 遠隔操作 | 目的達成 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 〇 |  |  |  |  |  |  |

::: message
サイバーキルチェーンとは、攻撃者がサイバー攻撃を成功させるまでの一連の流れを体系化したモデルです
:::

::: details ヘルプ
```
$ wafw00f -h
使用方法: wafw00f URL1 [URL2 [URL3 ... ]]
例: wafw00f http://www.victim.org/

オプション:
  -h, --help            このヘルプメッセージを表示して終了する。
  -v, --verbose         詳細出力を有効にする。-v を重ねる（例: -vv）ことで
                        詳細度を上げることができる。
  -a, --findall         シグネチャに一致するすべての WAF を検索する。
                        最初の一つが見つかってもテストを停止しない。
  -r, --noredirect      3xx レスポンス（リダイレクト）に従わない。
  -t TEST, --test=TEST  特定の WAF を指定してテストを行う。
  -o OUTPUT, --output=OUTPUT
                        指定した拡張子（csv, json, text）に応じてファイルに出力する。
                        標準出力にする場合はファイル名に - を指定する。
  -f FORMAT, --format=FORMAT
                        出力形式を csv, json, text のいずれかに強制する。
  -i INPUT, --input-file=INPUT
                        ファイルからターゲットを読み込む（csv, json, text）。
                        csv と json の場合、url という列名または要素が必要。
  -l, --list            WAFW00F が検出可能な WAF の一覧を表示する。
  -p PROXY, --proxy=PROXY
                        HTTP プロキシを使用してリクエストを行う。
                        (例: http://hostname:8080, socks5://hostname:1080,
                        http://user:pass@hostname:8080)
  -V, --version         現在の WAFW00F のバージョンを表示して終了する。
  -H HEADERS, --headers=HEADERS
                        デフォルトのヘッダーセットを上書きするため、
                        テキストファイルからカスタムヘッダーを読み込む。
  -T TIMEOUT, --timeout=TIMEOUT
                        リクエストのタイムアウト時間を設定する。
  --no-colors           出力の ANSI カラー（文字色）を無効にする。

```
:::

### 実行結果の見方

#### 調査対象

hxxps[://]www[.]sample[.]comに対して調査します
```
Checking https://www.sample.com
```

---

#### 特定したWAF製品

Cloudflare WAFを使用していることがわかりました
```
The site https://www.sample.com is behind Cloudflare WAF.
```

---

#### 送信したリクエスト

2つのリクエストで特定できました
```
Number of requests: 2
```

::: message
オプション-aを使用していないので1つ目のWAFが特定できた時点で終了しています
:::

### 結果の活用

各WAF製品には特長があり、既存の脆弱性が見つかっている可能性があります。
事前に対象が使用しているWAF製品がわかれば、わざわざいろんな攻撃を試す必要が無く、その製品の持つ脆弱性を調査すればよいので手間が省けます。
