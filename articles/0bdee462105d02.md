### dnsenum

dnsenumはドメイン名から関連するあらゆるDNS情報(A/MX/NSレコード)を引き出し、ターゲットのネットワーク構成を明らかにする列挙ツールです。
ターゲットがどのようなサブドメイン、メールサーバ、ネームサーバ、そしてIPアドレスの範囲を保有しているかを明らかにします。

基本的な使い方は以下
```
dnsenum sample.com
```

よく使うオプション

- -enum: ショートカットオプション。よく使うものが一括で実行できる。
- -f: 辞書攻撃で使用するワードファイル
- -thread: デフォルトだと遅いので10とか20にあげる

サイバーキルチェーンの「偵察」において使用されます。

| 偵察 | 武器化 | 配送 | 攻撃 | インストール | 遠隔操作 | 目的達成 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 〇 |  |  |  |  |  |  |

::: message
サイバーキルチェーンとは、攻撃者がサイバー攻撃を成功させるまでの一連の流れを体系化したモデルです
:::

::: details ヘルプ
```
$ dnsenum -h

使用方法: dnsenum [オプション] <ドメイン>
[オプション]:
注釈: -f タグを指定しない場合、デフォルトで /usr/share/dnsenum/dns.txt または dnsenum と同じディレクトリにある dns.txt ファイルが使用されます。
全般オプション:
  --dnsserver   <サーバー>
                        A、NS、MXレコードのクエリに使用するDNSサーバーを指定します。
  --enum                ショートカットオプション。--threads 5 -s 15 -w と同等です。
  -h, --help            このヘルプメッセージを表示します。
  --noreverse           逆引き（リバースルックアップ）操作をスキップします。
  --nocolor             ANSIカラー出力を無効にします。
  --private             実行結果の最後に出力される domain_ips.txt ファイルにプライベートIPを表示・保存します。
  --subfile <ファイル>   有効なすべてのサブドメインをこのファイルに書き込みます。
  -t, --timeout <値>    TCPおよびUDPのタイムアウト値を秒単位で指定します（デフォルト: 10秒）。
  --threads <値>        各種クエリを実行するスレッド数を指定します。
  -v, --verbose         詳細表示：すべての進行状況とエラーメッセージを表示します。
Googleスクレイピングオプション:
  -p, --pages <値>      名前をスクレイピングする際に処理するGoogle検索のページ数を指定します。デフォルトは5ページです。-s スイッチの指定が必須です。
  -s, --scrap <値>      Googleからスクレイピングするサブドメインの最大数を指定します（デフォルト: 15）。
ブルートフォースオプション:
  -f, --file <ファイル>  ブルートフォースを実行するために、このファイルからサブドメインを読み込みます（デフォルトの dns.txt より優先されます）。
  -u, --update  <a|g|r|z>
                        -f スイッチで指定したファイルを、有効なサブドメインで更新します。
        a (all)         すべての結果を使用して更新。
        g               Googleスクレイピング結果のみを使用して更新。
        r               逆引き結果のみを使用して更新。
        z               ゾーン転送結果のみを使用して更新。
  -r, --recursion       サブドメインに対する再帰実行。NSレコードを持つ、発見されたすべてのサブドメインに対してブルートフォースを繰り返します。
WHOISネットワーク範囲オプション:
  -d, --delay <値>   WHOISクエリ間の最大待ち時間を秒単位で指定します。値はランダムに決定されます（デフォルト: 3秒）。
  -w, --whois           クラスCのネットワーク範囲に対してWHOISクエリを実行します。
                        警告: 非常に広大なネットワーク範囲が生成される可能性があり、逆引きの実行に多大な時間を要する場合があります。
逆引きオプション:
  -e, --exclude <正規表現>
                        正規表現に一致するPTRレコードを逆引き結果から除外します。無効なホスト名を取り除くのに役立ちます。
出力オプション:
  -o --output <ファイル>    XML形式で出力します。MagicTree (www.gremwell.com) にインポート可能です。
```
:::

### 実行結果の見方

#### Host's addresses（Aレコード）

指定したドメイン自体のIPアドレス（Aレコード）が表示されます。

```
Host's addresses:
_________________
sample.com.                              5      IN    A        93.184.216.34
```

---

#### Mail Servers(MXレコード)

指定したドメイン宛てのメールを管理しているメールサーバーが表示されます

```
Mail (MX) Servers:
__________________
(No MX records found or sample output)
```

---

#### Name Servers（NSレコード）

指定したドメインを管理しているネームサーバーが表示されます。

```
Nameservers status:
____________________

a.iana-servers.net.                      AXFR record query failed: REFUSED
b.iana-servers.net.                      AXFR record query failed: REFUSED
```

- Trting Zone Transfers(ゾーン転送)

DNSサーバーは故障に備えてプライマリDNSとセカンダリDNSの2台以上で運用されます。
管理者によってプライマリDNSの情報が更新された場合、情報をコピーしてセカンダリDNSにも配布します。この配布して情報を同期することをゾーン転送と言います。

```
AXFR record query failed
```

::: message
攻撃者が、ゾーン転送を実行できた場合、ドメイン情報を全て入手することが可能になってしまいますが、普通はできません。
:::

---

#### Brute forcing (サブドメインの列挙)

指定された辞書ファイル(デフォルトは /usr/share/dnsenum/dns.txt)を使用して、よく使われるサブドメインが実在するかどうかを片っ端から問い合わせます。

```
Brute forcing with /usr/share/dnsenum/dns.txt:
_______________________________________________

www.sample.com.                          5      IN    A        93.184.216.34
mail.sample.com.                         5      IN    A        93.184.216.35
dev.sample.com.                          5      IN    A        93.184.216.36
```

---

#### 同一ネットワークの範囲

ターゲットのIPアドレスから推測した、同一ネットワーク上の範囲
```
sample.com class C netranges:
______________________________
93.184.216.0/24
```

---

#### 逆引き確認

ターゲットのIPアドレスから逆引きして特定したドメイン
```
Performing reverse lookup on netrange 93.184.216.0/24:
_______________________________________________________
34.216.184.93.in-addr.arpa.              5      IN    PTR      sample.com.
```

### 結果の活用

メールサーバの種類やバージョンから、脆弱性が見つかることがあります。
また、オープンリレーの設定が許可されていると、攻撃者は迷惑メールの踏み台として利用できます。
他にも、DKIM/SPF/DMARCが実装されていない場合、なりすましによるフィッシングメール攻撃の成功率が上がります。
取得したIPアドレスでターゲットに対してポートスキャンをすることで、公開されているサービスやバージョンを特定します。古いバージョンであれば脆弱性が見つかることがあります。
また、IPアドレスからドメインを逆引きし、一つのIPアドレスに複数のドメインが紐づいていれば、新たな攻撃対象になります。
サブドメインを列挙することで、古いアプリケーションやテスト環境、放置された脆弱なサーバが見つかることがあります。
