## セキュリティポリシー

セキュリティポリシーとは、経営陣による情報セキュリティに対する意思表明である。
組織として、何を、なぜ、どう守るのかを定義した文書である。

![](https://storage.googleapis.com/zenn-user-upload/1cfb7394e15e-20260202.png)

### 枠組み

セキュリティポリシーの枠組みとして大きく以下のように分けることができる

| 項目 | 必要性 | 内容 |
| ---- | ---- | ---- |
| ポリシー | 必須 | 方向性や目標|
| スタンダード | 必須 | ポリシーを実現するために必要な具体的な基準 |
| ベースライン | 任意 | スタンダードを決める根拠 |
| ガイドライン | 任意 | ポリシーを実現するための推奨事項 |
| プロシージャ | 必須 | ポリシーをどうやって達成するかを示す手順書 |


- スコーピング：スタンダードやベースラインの中からどれを採用するか選別すること
- テーラリング：スタンダードやベースラインを自社の環境に合わせてカスタマイズすること

### 具体例

ポリシーとして「社外に持ち出す全てのパソコンは、盗難や紛失による情報漏洩から保護しなければならない」と決まったとする。
スタンダードは「社外に持ち出す全てのパソコンのディスクは暗号化する」とし、ベースラインは国際規格に従ったためである。
ガイドラインは、「カフェや移動中はのぞき見防止フィルターを推奨」。
プロシージャは、デバイスの暗号化の詳細なマニュアル手順書。

:::message
セキュリティポリシーなどのルールは、必ず例外というものが発生する。その際に、誰がそのリスクを承認するのかなどをプロセス化しておくことも重要である。
:::

## 意思決定

意思決定には大きく「トップダウン」と「ボトムアップ」の２つが存在する。
セキュリティポリシーを策定する際は、基本的にはトップダウンにすべきである。
セキュリティポリシーは企業目標を達成するためのポリシーであり、経営戦略を理解している必要がある。
ボトムアップで決めたセキュリティポリシーは企業目標の達成を妨げる、現場優先のポリシーになる可能性がある。

:::message
情報漏洩が発生した場合、企業としては適切な注意（デューディリジェンス）を徹底できていたかが重要である。
ボトムアップで決めたポリシーをただ形として署名しただけでは、会社として十分なセキュリティ対策ができていたとは主張できなくなる。
:::

ただ、現場の声を完全に無視したようなポリシーになってしまい、業務効率を下げるようなことがあれば、企業が許可していない個人デバイスの使用（シャドーIT）が蔓延するような原因になる。
そのため、適度に現場からのフィードバックを吸い上げることが重要である。

:::message
社内のチャットツールのパスワード入力が面倒なので、LINEで連絡してしまう等
:::

## まとめ

- セキュリティポリシーにはポリシー、スタンダード、ベースライン、ガイドライン、プロシージャがある
- セキュリティポリシーは基本的にトップダウンだが、適度に現場のフィードバックを取り入れる
