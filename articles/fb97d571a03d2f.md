## リスクマネジメント

リスクマネジメントとは、将来起こりうるリスクを管理し、被害を最小限に抑えるためのプロセス。
リスク特定、リスク分析、リスク対応の3つのフェーズに分けられる。

![](https://storage.googleapis.com/zenn-user-upload/263e815935e2-20260206.png)

### リスク特定

どんなリスクが想定されるかを洗い出す。電子情報や音声情報など、デバイスに含まれるあらゆるデータの保護が必要である。

:::message
重要なのは資産の価値を定義することであり、価値が不明だと、どれだけの予算をかけるべきかが判断できない。
:::

- 資産の特定：守るべき資産（データ、ハードウェア、知的財産、人的資源）をリストアップする
- 脅威の特定：資産を脅かす要因（自然災害、サイバー攻撃、内部不正）を特定する
- 脆弱性の特定：資産に存在する弱点（パッチ未適用、従業員の教育不足）を特定する

$$リスク = 資産価値 × 脅威 × 脆弱性$$


脅威や脆弱性は時間とともに変化するものであり、年1回など定期的に最新情報を確認し情報を更新することが重要である。
また、リスクは脅威と脆弱性は単独ではリスクとは言えず、2つが重なって初めてリスクになる。

:::message
例えば、コンピュータは常にサイバー攻撃の脅威にさらされているが、電源を落としているのであれば脆弱性が無くなるのでリスクが無いと言える。ただし、盗まれるなど、他の脅威が存在することは忘れてはならない。
:::

### リスク分析

特定したリスクが企業にどの程度の影響を及ぼすかを評価、分析する。

#### 定量的分析

数値を使い、企業としての金銭的損失を算出する

- 年間予想損失額

SLE (Single Loss Expectancy): 1回の事故で失う金額。 
$$SLE = AV（資産価値） \times EF（暴露係数）$$

ARO (Annual Rate of Occurrence): 年間にその事故が起きる頻度。

ALE (Annualized Loss Expectancy): 年間の予想損失合計額。 
$$ALE = SLE \times ARO$$

::: message
暴露係数とは、1回の事故で損失する割合（0～100％）
:::

- 残留リスク

リスク対策を講じても、残ってしまうリスクのこと。

::: message
リスクをゼロにするのは難しいので、残ったリスクをどう扱うかが重要
:::

#### 定性的分析

数値を使って評価できないものに対して行う手法である。

例えば以下のようなものがあげられる。

- リスク分析表：発生頻度と、影響度を「高、中、低」などに分類して評価する。
- デルファイ法：専門家の意見を匿名で集約する
- アンケート：アンケートを取り、その結果から評価する

::: message
影響度の基準はいろいろあるが、以下は具体例
高：事業継続不可
中：手動での運用が不可で業務に大きな支障が出る
低：業務が多少不便になる程度
:::

### リスク対応

分析結果に基づき、どのように対処するかを決定する。

| 対応 | 内容 | 例 |
| ---- | ---- | ---- |
| 低減 | リスクを許容範囲内まで下げる | ファイアウォールの導入 |
| 移転 | リスクを他者に肩代わりさせる |サイバー保険への加入  |
| 回避 | リスクが高すぎる活動を中止する | 危険な地域での事業撤退 |
| 受容 | リスクが小さい、コストが大きい場合にそのまま受け入れる | 隕石が落ちた時のことは考えない |

![](https://storage.googleapis.com/zenn-user-upload/29048eb3b9c6-20260208.png)

## まとめ

- リスクマネジメントは「リスク特定 → 分析 → 対応」という一連のプロセスで構成される
- リスク分析には、金額で評価する定量的分析と、影響度や頻度で評価する定性的分析がある
- リスクはゼロにできないため、低減・移転・回避・受容のいずれかで対応し、被害を最小限に抑える
