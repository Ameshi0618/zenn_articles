### dnsmap

dnsmapは、ターゲットとなるドメインのサブドメインを列挙するツールです。

::: message
dnsenumはDNSゾーン転送やWhois情報の取得など、より多機能で広範囲な情報を取得できましたが、dnsmapはサブドメインの総当たりに特化した、シンプルで軽量なツールです。
:::

基本的な使い方は以下
```
dnsmap sample.com
```

よく使うオプション

- -w: 使用するワードリストファイル

サイバーキルチェーンの「偵察」において使用されます。

| 偵察 | 武器化 | 配送 | 攻撃 | インストール | 遠隔操作 | 目的達成 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 〇 |  |  |  |  |  |  |

::: message
サイバーキルチェーンとは、攻撃者がサイバー攻撃を成功させるまでの一連の流れを体系化したモデルです
:::

::: details ヘルプ
```
$ dnsmap                
dnsmap 0.36 - DNS Network Mapper

使用法: dnsmap <攻撃対象ドメイン> [オプション]

オプション:
-w <ワードリストファイル>
-r <通常の実行結果ファイル>
-c <CSV形式の実行結果ファイル>
-d <遅延>
-i <無視するIPアドレス> (誤検知が発生する場合に役立ちます)

実行例:
dnsmap example.com
dnsmap example.com -w yourwordlist.txt -r /tmp/domainbf_results.txt
dnsmap example.com -r /tmp/ -d 3000
dnsmap example.com -r ./domainbf_results.txt
```
:::

### 結果の活用

メールサーバの種類やバージョンから、脆弱性が見つかることがあります。
また、オープンリレーの設定が許可されていると、攻撃者は迷惑メールの踏み台として利用できます。
他にも、DKIM/SPF/DMARCが実装されていない場合、なりすましによるフィッシングメール攻撃の成功率が上がります。
取得したIPアドレスでターゲットに対してポートスキャンをすることで、公開されているサービスやバージョンを特定します。古いバージョンであれば脆弱性が見つかることがあります。
また、IPアドレスからドメインを逆引きし、一つのIPアドレスに複数のドメインが紐づいていれば、新たな攻撃対象になります。
サブドメインを列挙することで、古いアプリケーションやテスト環境、放置された脆弱なサーバが見つかることがあります。
