## セキュリティ管理フレームワーク

前回の記事（CISSPで学ぶ「セキュリティガバナンス2」）で2つの有名なフレームワーク（ISO/IEC 27000シリーズ、NIST SP800-39）を紹介したが、他にもいくつか存在する。

### ・COBIT

ISACA（情報システムコントロール協会）とITGI（ITガバナンス協会が）が策定した、ITガバナンスとITマネジメントのための包括的なフレームワークである。

ガバナンスと情報技術は切り離せないものであり、両者の関係を整理し、リスク管理ができるようにすることが目的である。

| レベル | 名称 | 状態 |
| ---- | ---- | ---- |
| 0 | Incomplete（欠如） | リスクを認識できていない |
| 1 | Initial（初期） | リスクは認識しているが、場当たり的な対応 |
| 2 | Managed（管理可） | リスクは管理されているが、プロセスは組織で統一できていない |
| 3 | Defined（定義済） | リスクが管理され、プロセスが組織で統一できている。プロセス改善はしていない |
| 4 | Quantitative（定量的に管理可） | リスクが管理され、プロセスが組織で統一できている。プロセス改善はしているがまだ改善の余地がある |
| 5 | Optimizing（最適化） | リスク管理、プロセスが最適化されている |

COBITで組織を評価する際、4つの軸が存在する。

| 軸 | 名称 | 内容 |
| ---- | ---- | ---- |
| 計画と組織 | Plan and Organize (PO) | 企業がビジネス目標を達成するためにITをどう活用するか、戦略、組織体制、投資計画などを策定する段階 |
| 調達と導入 | Acquire and Implement (AI) | 戦略に基づき、具体的なシステムやソフトウェアを開発、調達し、それをビジネスプロセスに統合する段階 |
| デリバリとサポート | Deliver and Support (DS) | 開発、導入したシステムを実際に運用し、ユーザーへのサービス提供やセキュリティの維持、トラブル対応などを行う段階 |
| モニタリングと評価 | Monitor and Evaluate (ME) | ITプロセスが正しく機能しているか目標を達成しているかを継続的に監視、評価する段階 |

### ・COSO

組織が目的を達成するために必要な「内部統制」のあり方を定義しており、粉飾決算などの不正を防ぐためのものである。

:::message
粉飾決算とは、企業の行政を示す財務諸表と実態が異なることを指す。財務諸表は投資家や銀行が、「その企業にお金を出していいか」を判断する材料なので、それが正しくなければ正しい投資判断ができません。また最悪の場合、企業の信頼を失い倒産に至るケースもある。
:::

#### COSOキューブ

内部統制が「どのような目的で」「どのような要素で構成され」「組織のどこに適用されるか」という3つの側面の相互関係を示している。

- 上面：組織が内部統制によって達成しようとする目標
- 左側面：目的達成のために必要な仕組み。下から順に積む上がっている。
- 右側面：内部統制が適用される組織の範囲やレベル

上面の目的を達成するためには、左側面の5つの内部統制のための構成要素が正しく積み上がっており、右側面の全ての範囲において内部統制が適用されている必要がある。

### ・PCI DSS

クレジットカードなどの電子決済を行う際の個人情報流失を防ぐためのフレームワーク。カードデータを暗号化したり、データへのアクセスを必要最小限に絞るなど様々の規定が定められている。

クレジットカード情報はサイバー犯罪者にとって価値の高い物であり、カード情報が流出し、不正利用されてしまえば被害額は甚大になる。

## ITIL

ITサービスマネジメントにおけるベストプラクティスを集めたガイドブック。ベストプラクティスとは、最適化されたものではないが、実績に基づいて「最も効果的」と考えられる具体的な方法や考え方のこと。有名なPDCAサイクルもベストプラクティスの一つである。

## まとめ

- COBITは企業のITガバナンスの成熟度を評価するフレームワーク
- COSOは内部統制のためのフレームワーク
- PCI DSSは電子決済の個人情報保護のためのフレームワーク
- ITILはITサービスマネジメントにおけるベストプラクティス集
