## アクセス制御

データにはアクセスする側とされる側が存在する。
「誰が・何に・どこまでアクセスしてよいか」を定め、認可および禁止をする仕組みのことである。

![](https://storage.googleapis.com/zenn-user-upload/25d3f11cc0a0-20260118.png)

| 要素 | 説明 |
| ---- | ---- |
| 主体（subject） | 人間やコンピュータなど、データにアクセスする側を表す概念である。 |
| 対象（object） | データやドキュメントなど、アクセスされる側を表す概念である。 | 

### 最小権限(Least Privilege)の原則

権限は業務を遂行するのに必要な必要『最小限』のみの権限を与えるという設計思想である。
最小権限は人間依存の考えで、「この人にはこの権限がある」という考え。

- 目的：攻撃者がアカウントを乗っ取った場合の被害を最小化する
- 対象：人間だけでなく、プログラムやデバイスアカウントなどすべてに適応される

### 知る必要性(Need-to-Know)

機密性が高い情報には、権限が必要な者に必要な『最短期間』だけ与える、という設計思想である。
知る必要性はタスク依存の考えで、「このタスクで必要な情報はこれだからその権限を与える」という考え。

- 目的：情報漏洩の防止。機密情報をできるだけ機密にし続けること。
- 対象：フォルダ、ファイル、データベース内の特定のレコードなど

:::message
データベース管理者であれば、データベースへの管理のみを許可する（最小権限）。
ただし、データベースの一部は編集の必要がないため閲覧不可にする（知る必要性）。
:::

## 真正性と否認防止

何かを守る際には、情報にアクセスしてきた人が本人であることや、ある主体の行動が否定できないようにすることが重要である。

### 真正性

十分に認証することで本人であることを保証すること。
真正性が保証されていない場合は、なりすましによってデータが侵害される場合がある。

### 否認防止

ある人の行為が追跡可能であること。つまり、「そんなことはやっていない」と言わせないことである。
否認防止ができていないケースとして、ログを取得していないなどが考えられる。

:::message
複数人で同一アカウントを使用している場合も、否認防止ができない状況である。
:::

## まとめ

- アクセス制御は最小権限と知る必要性の2つの観点で考えることが重要
- 真正性とは、ある対象が本もであることを保証すること
- 否認防止とは、ある主体の行動に対して否定できないようにすること
