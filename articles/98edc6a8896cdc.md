## セキュリティ管理策

セキュリティリスク管理策は大きく3つに分類される。

![](https://storage.googleapis.com/zenn-user-upload/6139fcc89dd0-20260210.png)

### 管理的対策

人や組織の運用に焦点を当てた対策である。ポリシーやルール、管理上の手続きを指します。

- 定義：セキュリティに関する意思決定、手順、ガイドラインの策定。
- 具体例：セキュリティポリシーの策定、従業員のセキュリティ意識向上トレーニング、採用時のバックグラウンドチェック、職務分離

### 技術的対策

システムやソフトウェアに実装される対策である。

- 定義：情報システムやネットワーク内で動作する論理的な制限
- 具体例：アクセス制御、ファイアウォール、暗号化、アンチウイルスソフト導入

### 物理的対策

物理的に存在する機材を守るための対策である。施設への侵入や、盗難、破壊から保護する。

- 定義：建物、周辺環境、機材その真野に対する物理的な障壁
- 具体例：入退室管理、防犯カメラの設置、防火システム、フェンス

---

### セキュリティ管理策のカテゴリ

セキュリティ管理策はその機能によって以下の７つに分類できる。

| 機能 | 目的 | 具体例 |
| ---- | ---- | ---- |
| 抑止 | 攻撃者にやめようと思わせる（心理的） | 警告看板、警備犬 |
| 防止 | 攻撃のやりづらさを上げる | ファイアウォール、フェンス |
| 検知 | 発声したインシデントに気づく | ログ監視、IDS |
| 是正 | 障害の再発防止 | パッチ適用、マルウェア駆除 |
| 回復 | 障害発生後に復旧を行う | バックアップ |
| 指示 | 従業員への行動指示 | 教育 |
| 補完 | 他の管理策では不十分な部分を補う | ドローン監視 |

:::message
1つの対策が「複数の機能を持つことがある」という点が重要である。例えば監視カメラが見える位置にあった場合、抑止的であり、かつ検知的、防止的でもある。
:::

## 継続的モニタリング

定期的な確認ではなくリアルタイムでリスクを評価し続けることである。
攻撃手法や脆弱性は毎日更新されており、昨日の安全は今日の安全とは限りません。

## SOCレポート

SOCレポートとは、受託会社が委託会社に対して、自社の内部統制、セキュリティが適切に機能しているかを証明するためのレポートであり、監査法人や公認会計士が客観的に評価して作成する。
米国公認会計士協会（AICPA）が定めたSSAE18という監査基準によって規定されています。

:::message
監査において重要なのは客観性である。自社の企業の内部統制を自社で評価しても甘くなってしまったりするため意味がない。
:::

SOCレポートにはSOC1、SOC2、SOC3の3種類がある。

| 種類 | 保証対象 | 用途 | 閲覧制限 |
| ---- | ---- | ---- | ---- |
| SOC1 | 財務報告に関連する内部統制 | 財務諸表監査の効率化 | 限定（既存顧客、監査人） |
| SOC2 | セキュリティなど | 情報セキュリティの妥当性確認 | 限定（既存顧客、監査人） |
| SOC3 | （SOC2の要約） | マーケティング・一般公開用 | なし（誰でも閲覧可） |

![](https://storage.googleapis.com/zenn-user-upload/2e051c775289-20260210.png)

### Trustサービス基準（TSC）

SOC2では5つの評価対象が存在する

- セキュリティ：不正アクセスの防止
- 可用性：システムの稼働時間
- 処理の整合性：データの正確性、ラグのない処理
- 機密性：機密情報の保護
- プライバシー：個人情報の収集、使用、開示、廃棄

:::message
ベンダー選定において、セキュリティ評価をするのに元も重要なSOCレポートはSOC2レポートである
:::

## まとめ

- セキュリティ管理策は管理的、技術的、物理的の3つに分類できる
- SOCレポートは企業の内部統制を保証するものである
- セキュリティの評価においてはSOC2が重要である
