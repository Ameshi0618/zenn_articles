## セキュリティと法律

CISSPとしてのセキュリティの第一目標は「企業目標の達成」である。しかし、企業目標のためにセキュリティを怠ってはならない。
例えば、コスト削減のために脆弱な製品を販売したり、利益のために個人情報を売ったりしてはならない。

:::message
2024年に施行されたEUサイバーレジリエンス法（CRA）の罰則金は「最大1,500万ユーロ（約24億円）」または「全世界の年間売上高の2.5%」のいずれか高い方が科される。一時の利益のための行いの先には、企業の存続を揺るがすほどの損失が待っていることを忘れてはならない。
:::

![](https://storage.googleapis.com/zenn-user-upload/c24f260033cd-20260124.png)

### 法改正

法改正が行われた際には、企業のセキュリティポリシーや情報システムの再構築が必要になる。
新しい法律ができたということは、それに従った適切なデューケアが求められる。古いセキュリティポリシーや情報システムのまま、万が一セキュリティインシデントが発生した場合は「必要な対策を怠った」として法的責任を問われるリスクが生じる。
また、法改正直後は前例がないため、法律を満たす要件を考えることは難しくなる

:::message
CRAなどの法律は詳細な機能要求ではなく設計思想であるため、開発、設計の早い段階から取り込むことが重要。
:::

## 米国法

CISSPはアメリカの資格のため、米国の法律を覚える必要がある。
米国法を理解するうえでまずは米国の法体系を理解する必要がある。

| 分類 | 目的 | 立証基準 |
| ---- | ---- | ---- |
| 刑法 | 社会秩序の維持、犯罪への罰則 | ほぼ100%の確信が必要 |
| 民法 | 個人や組織間のトラブル、被害の救済 | 51％以上の正しさが必要 |
| 行政法 | 政府機関が定める規則や運用 | 明確で説得力があること |

刑事裁判における判断基準として**beyond a reasonable doubt**という言葉がある。これは、真犯人や、証拠の捏造など、少しでも疑いが残っているものがあれば有罪にしてはいけないというものである。

それに対して、民法は証拠の優位性が認められれば勝訴できる。

:::message
判断基準の違いによって次のようなことが起こりえる。
アメリカで起きた殺人事件（O.J.シンプソン事件）で、刑事裁判では100%の確信が得られなかっため無罪となったが、その後民事訴訟を起こされた結果、死亡について責任があるという判決で多額の賠償命令の支払いを命じられた。
:::

## 情報セキュリティ関連

### CFAA（コンピュータ詐欺と濫用に関する法律）

1986年に制定されたハッキング対策を目的としたアメリカの法律である。

元々、政府機関や金融機関のコンピュータのみが対象とされていたが、その後の法改正法により、インターネットに接続されているほぼすべてのデバイスが保護対象となった。

CFAAの体系は以下の7つのカテゴリーの行為を犯罪としている

| 条項 | 内容 | 説明 |
| ---- | ---- | ---- |
| 1 | 国家安全保障情報の取得 | 国防や外交に関する情報を不正に取得し、国家に損害もしくは外国の利益のために利用する行為 |
| 2 | 情報の不正取得 | 金融機関、政府機関、または「保護されたコンピューター」から情報を取得する行為 |
| 3 | 政府コンピューターへの不法侵入 | 米国政府のコンピューターへ許可なく立ち入る行為 |
| 4 | 詐欺目的のアクセス | 詐欺を働き、価値のあるものを入手する目的でアクセスする行為 |
| 5 | 損害を与える行為 | ウイルス（マルウェア）の送信や、システムの破壊・改ざん |
| 6 | パスワードの売買 | コンピューターへのアクセスに使用するパスワード等の取引 |
| 7 | 恐喝 | システムへのダメージや情報の漏洩を盾に金品を要求する行為（ランサムウェア等） |

:::message
CISSPとして、相手を説得するには法律の知識だけでは「本当に問題？」「そこまでする必要ある？」と思われる場面が出てくるかもしれない。そのためにも事例を知っておく必要があり、そうすることでリスクが現実味を帯び、説得力も増した説明が可能になる。
:::

#### Van Buren事件

「許可されたアクセスを超える」の定義を扱ったアメリカの事件である。

警察官が知人から「このナンバー、誰の車か調べてくれたら金払うよ」と提案され、自身の権限を利用してデータベースにアクセスし情報を提供した事件ある。

- 従来：アクセス権はあるが目的が不適切な場合はCFAA違反。
- 現在：アクセス権があれば、目的が不適切でもCFAA違反にはならない。権限がない場所にアクセスした場合のみ違反。

ただ、CFAA違反ではないだけで別の罪や懲戒処分（汚職、職権乱用など）の対象になる可能性はある。

:::message
不正アクセスは罰するためには、正しくアクセスコントロールをする必要がある
:::

#### Sandvig vs Barr

ウェブサイトの利用規約違反を犯罪としないという判決を下した事例。

ウェブサイトの監査をする目的で架空のアカウントを作成し調査をする行為がCFAA違反である「許可なく、または許可された範囲を超えてアクセスすること」に当たるのではないかという指摘。
判決としては、ウェブサイトの利用契約は一方的に更新されることもあり、常に規約を守り続けることは難しく、それを違反というのはやりすぎなため**CFAAに違反しない**とした。

### CCPA（カリフォルニア州消費者プライバシー権法）

CCPAはプライバシー保護を求める法律であり、2020年から施行されている。
2023年にCPRA（カリフォルニア州プライバシー権法）がCCPAをアップデートする形で施行された。

:::message
CCPAが無くなったわけではなく、パッチのようなイメージ。
また、CCPA/CRPAは連邦法ではなく州法のため「連邦法＞州法」という力関係も意識しておきたい。
:::

消費者に与えられる権利として以下のものがある

| 権利 | 内容 | 
| ---- | ---- |
| 知る | 企業が収集、共有、販売した情報の内容や目的を開示させる権利 |
| 削除 | 企業が収集した個人情報を削除させる権利 |
| 訂正 | 企業が収集した個人情報が不正確な場合訂正させる権利 |
| オプトアウト | 企業が収集した個人情報を販売、共有することを拒否できる権利 |
| 制限 | 機密性の高い個人情報の使用目的を制限させる権利　※CPRAで新設 |
| 未成年者のオプトイン | 16歳未満の未成年者の個人情報の販売、共有についてオプトインを要求する権利 |
| 平等 | 消費者の権利をしたことを理由に、不当な扱いを受けない権利 |

- オプトアウト： デフォルトが同意で、拒否したい場合は自ら申し出ること。
- オプトイン： デフォルトが拒否で、事前にユーザーの同意を得る必要がある。

オプトは選ぶという意味なので、ユーザーが同意（イン）をすることを選ぶのか、拒否（アウト）することを選ぶのか、で覚えるとよいだろう。

:::message
Webサイトを訪れた際にクッキーの使用許可を確認される場合があるが、それがオプトインである
:::

#### ダークパターン規制違反

ダークパターンとはウェブサイトやアプリのデザインや表示を悪用し、ユーザーを誤解させたり、不利な選択をさせる手法である。
2025年3月12日、カリフォルニア州のプライバシー保護機関（CPPA）は、アメリカ・ホンダがCCPA違反したとして約63万ドルの罰金の支払いを命じた。
これは、消費者が個人情報の共有を拒否するためのオプトアウトの手続きを複雑にしたり、過剰の情報を要求したことが問題視された。

:::message
オプトインするための手続きが1クリックに対して、オプトアウトするためには複数回クリック複数回画面遷移、複数の個人情報提出などが求められることは明らかに非対称であり問題である。
:::

### FISMA（連邦情報セキュリティマネジメント法）

米国の政府機関とその関係者がどのように情報を守るべきかを義務付けた法律である。
FISMAを順守するためのフレームワークがNIST SP 800-37である。
