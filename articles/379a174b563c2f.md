## 情報と資産

情報とは、意味を持つデータのことである。業務において使用した情報や画像、会議の録音などさまざまのものが含まれる。
資産とは、組織にとって価値のあるものすべてを指すものである。デジタル情報だけでなく、物理的に存在するパソコンやサーバーや、従業員、会社としてのブランド価値などを含み、情報よりも広い範囲を指す。
:::message
情報は資産の一部
:::

## ラベリング

ラベリングはアクセス制御における手法の一つであり、操作対象となる資産の重要度に応じて適切な保護を適用する。
適切なラベリングを行うことで、効率的に資産を保護し、組織のリスクを最小限にすることができる

:::message
なんでもかんでも保護レベルを高くしてしまうと、作業効率が低下するので注意が必要
:::

![](https://static.zenn.studio/user-upload/4f4e4e551b0d-20260704.png)
### ラベリングの流れ

ラベリングは以下の順番で実施する

| No | 項目 | 概要 |
| ---- | ---- | ---- |
| ➀ | 資産の特定 | 組織が持つ資産を洗い出す |
| ➁ | 資産の評価 | 各資産の重要度を評価する |
| ➂ | ラベル検討 | 漏洩時の影響度によって分類し、必要なラベルの種類を検討する |
| ➃ | ラベリングの実施 | 各資産のラベリングを行う |
| ⑤ | 維持改善 | ラベリングが正しく実施されているか管理し、必要に応じて改善する |

### ラベリング方法

ラベリングの方法には、物理的なものと電子的ものが存在する。

物理的なものとしては、物理的な資産に直接シールなどを張り付けることなどが含まれる。
電子的なものとしては、重要なファイルのヘッダーやフッターなどに「社外秘」、「機密」などを書いておくことが含まれる。

電子的なラベリングのメリットとしてはDLP（データ損失防止）のシステムなどによって、社外に機密情報などをメールを送信しようとした場合に文字列を検知し、情報漏洩の防止ができることなどがあげられる。

:::message
ラベリングが変更される例として、新商品に関する情報が考えられます。情報のリリース前は機密事項であっても、リリース後は機密レベルが下がると考えられる。
:::

## クリアランス

ここまではアクセスされる「資産側」の管理について述べてきましたが、アクセスする「ユーザー側」の管理も不可欠である。資産の重要度（ラベル）に応じ、ユーザーに対して必要なアクセス権限を付与することを「クリアランス」という。
